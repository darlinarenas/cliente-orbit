MASTER ORBIT ASSISTANT PRO PREMIUM
Proyecto completo escalable - Vista cliente + administrador + backend + IA + QR + multimedia

Versión: 1.0
Objetivo comercial: producto premium vendible a empresa por USD 20.000+
Marca del sistema: Orbit Assistant
Enfoque: asesor digital de productos Orbit para retail, con QR por producto, ficha técnica, recomendaciones cruzadas, guía de instalación, videos, IA y panel administrador.


============================================================
1. VISIÓN GENERAL DEL PRODUCTO
============================================================

Orbit Assistant será una plataforma web completa donde cada producto Orbit tendrá un QR único.

Cuando un cliente escanee el QR desde una tienda como Sodimac, Easy, Imperial, Construmart u otro retail, verá una experiencia móvil premium:

- Producto detectado.
- Ficha técnica simple y visual.
- Guía de instalación.
- Videos o video-fotos de instalación.
- Preguntas frecuentes.
- Productos compatibles.
- Recomendaciones cruzadas para aumentar ventas.
- Chat con Orbit Assistant.
- Detalle navegable de productos recomendados.
- Sugerencias según tipo de proyecto: jardín pequeño, mediano, grande, riego automático, césped, maceteros, huerto, etc.

El administrador podrá:

- Crear productos.
- Editar productos.
- Subir imágenes.
- Subir videos.
- Crear guías de instalación.
- Asociar productos compatibles.
- Crear recomendaciones inteligentes.
- Generar QR por producto.
- Ver métricas de escaneos.
- Ver preguntas frecuentes realizadas por clientes.
- Administrar categorías, líneas y fichas técnicas.
- Activar o desactivar productos.
- Administrar contenido para IA.
- Revisar estadísticas por producto, tienda, región y fecha.

2.1 MÓDULO CLIENTE QR
------------------------------------------------------------

Vista pública que ve el cliente al escanear un QR.

Funciones:

- Detectar producto por slug o código QR.
- Mostrar pantalla de bienvenida con mascota Orbit Assistant.
- Mostrar producto escaneado.
- Mostrar ficha técnica resumida.
- Mostrar ficha técnica completa.
- Mostrar imágenes del producto.
- Mostrar video de instalación.
- Mostrar guía paso a paso.
- Mostrar productos compatibles.
- Mostrar productos recomendados para completar instalación.
- Permitir entrar al detalle de cualquier producto recomendado.
- Permitir preguntas rápidas.
- Permitir chat libre con Orbit Assistant.
- Mostrar advertencias o consejos:
  - "Para este producto también podrías necesitar..."
  - "No olvides revisar la presión de agua."
  - "Este accesorio mejora la instalación."
  - "Este producto funciona mejor con programador."
- Mostrar modo proyecto:
  - "Quiero regar césped."
  - "Quiero automatizar el riego."
  - "Tengo un jardín pequeño."
  - "Tengo un jardín de 50 m²."
  - "Quiero instalar riego por goteo."

Pantallas cliente:

- /p/:slug
- /p/:slug/ficha
- /p/:slug/instalacion
- /p/:slug/compatibles
- /p/:slug/chat

2.7 MÓDULO CHAT ORBIT ASSISTANT
------------------------------------------------------------

El chat debe ser útil para cliente final, no técnico.

Funciones:

- Preguntas rápidas.
- Pregunta libre.
- Respuestas simuladas al inicio.
- Luego integración con IA real.
- Contexto del producto escaneado.
- Contexto de productos recomendados.
- Contexto de ficha técnica.
- Respuestas comerciales inteligentes.
- Recomendaciones cruzadas.
- Sugerencia de instalación completa.
- Lenguaje simple.

Ejemplos de preguntas:

- ¿Sirve para césped?
- ¿Qué presión necesita?
- ¿Qué necesito para instalarlo?
- ¿Necesito programador?
- Tengo un jardín de 50 m², ¿qué compro?
- ¿Este producto sirve con una boquilla?
- ¿Cuál es compatible?
- ¿Cómo se instala?
- ¿Cuántos necesito para mi jardín?
- ¿Qué producto me falta?

El asistente debe responder:

- Claro.
- Corto.
- Comercial.
- Con recomendaciones.
- Sin inventar datos técnicos que no existan.
- Si no sabe, debe decir que necesita revisar ficha o recomendar consultar a un especialista.

7. FLUJO COMPLETO DEL CLIENTE
============================================================

Flujo principal:

1. Cliente ve producto en tienda.
2. Cliente escanea QR.
3. Sistema abre /q/:qrCode.
4. Backend registra escaneo.
5. Backend resuelve producto.
6. Frontend muestra landing del producto.
7. Cliente ve:
   - Producto.
   - Ficha rápida.
   - Beneficios.
   - Botón "Completar instalación".
   - Botón "Preguntar a Orbit Assistant".
8. Cliente entra a recomendados.
9. Sistema muestra accesorios necesarios.
10. Cliente toca un producto recomendado.
11. Sistema abre detalle del producto recomendado.
12. Cliente puede volver al producto original.
13. Cliente puede ver guía de instalación.
14. Cliente puede reproducir video.
15. Cliente puede preguntar al asistente.
16. Chat responde usando contexto del producto.
17. Se registra la pregunta en el backend.
18. Admin luego puede ver métricas y preguntas frecuentes.


============================================================
8. FLUJO COMPLETO DEL ADMINISTRADOR
============================================================

1. Admin entra a /admin/login.
2. Inicia sesión.
3. Ve dashboard.
4. Crea categoría si no existe.
5. Crea producto.
6. Sube imagen principal.
7. Sube galería.
8. Llena ficha técnica.
9. Agrega manual PDF.
10. Agrega video de instalación.
11. Crea pasos de instalación.
12. Asocia productos compatibles.
13. Asocia productos recomendados.
14. Define tipo de recomendación:
    - Necesario.
    - Recomendado.
    - Premium.
    - Opcional.
15. Genera QR.
16. Descarga QR.
17. Activa producto.
18. Revisa vista pública.
19. Revisa escaneos.
20. Revisa preguntas.
21. Ajusta recomendaciones.


============================================================
9. REGLAS DE NEGOCIO
============================================================

Reglas importantes:

- Un producto puede tener muchos productos recomendados.
- Un producto puede ser recomendado por muchos productos.
- Un producto puede tener muchos QR.
- Un QR puede tener información de tienda o campaña.
- Un producto puede tener muchas imágenes.
- Un producto puede tener un video principal.
- Un producto puede tener muchos pasos de instalación.
- Una pregunta siempre debe guardarse.
- Un producto inactivo no debe aparecer públicamente.
- Un QR inactivo debe mostrar mensaje de QR no disponible.
- El cliente no debe ver precios si Orbit no lo autoriza.
- La IA no debe inventar ficha técnica.
- La IA debe sugerir accesorios compatibles.
- Las recomendaciones deben poder ordenarse por prioridad.
- El administrador debe poder cambiar todo sin tocar código.


============================================================
10. ESCALABILIDAD
============================================================

El sistema debe estar preparado para:

- 20 productos demo.
- 200 productos iniciales.
- 2.000 productos reales.
- Varias categorías.
- Varias tiendas.
- Varias regiones.
- Miles de escaneos.
- Muchas preguntas de IA.
- Videos e imágenes por producto.
- Múltiples administradores.
- Nuevas marcas en el futuro si Vexora vende el sistema a otros clientes.

Recomendaciones técnicas:

- Frontend: React + Vite.
- Backend: Node.js + Express.
- Base de datos: PostgreSQL.
- Hosting frontend: Vercel.
- Hosting backend: Render.
- Base de datos: Supabase PostgreSQL.
- Archivos: Cloudinary, Supabase Storage o S3.
- IA: OpenAI API.
- QR: librería qrcode.
- Auth: JWT.
- Validaciones: Zod o Joi.
- Uploads: Multer.
- Seguridad: Helmet, CORS, Rate Limit.
